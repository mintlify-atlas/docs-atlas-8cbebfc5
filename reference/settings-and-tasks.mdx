title: Settings and Tasks
description: Reference for sbt settings and tasks including key types, common operations, and usage patterns
---

sbt uses a key-value model where **settings** are evaluated once at project load time and **tasks** are evaluated on demand. This page documents the core settings and tasks available in sbt.

## Key Types

sbt defines three main types of keys:

| Key Type | Description | Evaluation |
|----------|-------------|------------|
| `SettingKey[T]` | Values computed once at project load | Once per session |
| `TaskKey[T]` | Executable tasks that may have side effects | On demand |
| `InputKey[T]` | Tasks that accept command-line input | On demand with args |

## Core Build Settings

### Project Identity

<ParamField path="name" type="SettingKey[String]" default="project directory name">
  Project name used for artifact publication and display.
  
  ```scala
  name := "my-project"
  ```
</ParamField>

<ParamField path="organization" type="SettingKey[String]">
  Organization or group ID for the project.
  
  ```scala
  organization := "com.example"
  ```
</ParamField>

<ParamField path="version" type="SettingKey[String]">
  Project version for artifact publication. Default: `"0.1.0-SNAPSHOT"`
  
  ```scala
  version := "1.0.0"
  ```
</ParamField>

### Scala Configuration

<ParamField path="scalaVersion" type="SettingKey[String]">
  The version of Scala used for building. This is one of the most important settings.
  
  ```scala
  scalaVersion := "3.3.1"
  ```
</ParamField>

<ParamField path="scalaBinaryVersion" type="SettingKey[String]">
  The Scala version substring describing binary compatibility (e.g., "2.13", "3").
  Automatically derived from `scalaVersion`.
</ParamField>

<ParamField path="crossScalaVersions" type="SettingKey[Seq[String]]">
  The versions of Scala used when cross-building.
  
  ```scala
  crossScalaVersions := Seq("2.13.12", "3.3.1")
  ```
</ParamField>

<ParamField path="scalaOrganization" type="SettingKey[String]">
  Organization/group ID of the Scala used in the project. Advanced setting for Scala language clones. Default: `"org.scala-lang"`
</ParamField>

### Path Settings

<ParamField path="baseDirectory" type="SettingKey[File]">
  The base directory for the build, project, configuration, or task depending on scope.
  
  ```scala
  sourceDirectory := baseDirectory.value / "src"
  ```
</ParamField>

<ParamField path="target" type="SettingKey[File]">
  Main directory for files generated by the build.
  
  ```scala
  // Default: baseDirectory.value / "target"
  target := baseDirectory.value / "output"
  ```
</ParamField>

<ParamField path="sourceDirectory" type="SettingKey[File]">
  Default directory containing sources.
  
  **Default:** `baseDirectory.value / "src"`
</ParamField>

<ParamField path="scalaSource" type="SettingKey[File]">
  Default Scala source directory.
  
  **Default:** `sourceDirectory.value / "scala"`
</ParamField>

<ParamField path="javaSource" type="SettingKey[File]">
  Default Java source directory.
  
  **Default:** `sourceDirectory.value / "java"`
</ParamField>

<ParamField path="resourceDirectory" type="SettingKey[File]">
  Default unmanaged resource directory for user-defined resources.
  
  **Default:** `sourceDirectory.value / "resources"`
</ParamField>

<ParamField path="classDirectory" type="SettingKey[File]">
  Directory for compiled classes and copied resources.
  
  **Default:** `target.value / "classes"`
</ParamField>

## Core Tasks

### Compilation Tasks

<ParamField path="compile" type="TaskKey[CompileAnalysis]">
  Compiles sources and returns compilation analysis.
  
  ```bash
  sbt compile
  sbt Test/compile
  ```
</ParamField>

<ParamField path="clean" type="TaskKey[Unit]">
  Deletes files produced by the build, such as generated sources, compiled classes, and task caches.
  
  ```bash
  sbt clean
  ```
</ParamField>

### Interactive Tasks

<ParamField path="console" type="TaskKey[Unit]">
  Starts the Scala interpreter (REPL) with the project classes on the classpath.
  
  ```bash
  sbt console
  ```
</ParamField>

<ParamField path="consoleQuick" type="TaskKey[Unit]">
  Starts the Scala interpreter with the project dependencies on the classpath (without project classes).
  
  ```bash
  sbt consoleQuick
  ```
</ParamField>

### Run Tasks

<ParamField path="run" type="InputKey[Unit]">
  Runs a main class, passing along arguments provided on the command line.
  
  ```bash
  sbt run
  sbt "run arg1 arg2"
  ```
</ParamField>

<ParamField path="runMain" type="InputKey[Unit]">
  Runs the main class selected by the first argument, passing the remaining arguments to the main method.
  
  ```bash
  sbt "runMain com.example.Main arg1 arg2"
  ```
</ParamField>

### Test Tasks

<ParamField path="test" type="InputKey[TestResult]">
  Executes the tests that either failed before, were not run, or whose transitive dependencies changed.
  
  ```bash
  sbt test
  ```
</ParamField>

<ParamField path="testOnly" type="InputKey[TestResult]">
  Executes the tests provided as arguments or all tests if no arguments are provided.
  
  ```bash
  sbt "testOnly com.example.MySpec"
  ```
</ParamField>

<ParamField path="testQuick" type="InputKey[TestResult]">
  Executes tests that failed in previous runs or whose dependencies changed.
  
  ```bash
  sbt testQuick
  ```
</ParamField>

### Package Tasks

<ParamField path="package" type="TaskKey[File]">
  Produces the main artifact, such as a binary JAR. Typically an alias for `packageBin`.
  
  ```bash
  sbt package
  ```
</ParamField>

<ParamField path="packageBin" type="TaskKey[File]">
  Produces a main artifact, such as a binary JAR.
  
  ```bash
  sbt packageBin
  ```
</ParamField>

<ParamField path="packageSrc" type="TaskKey[File]">
  Produces a source artifact containing sources and resources.
  
  ```bash
  sbt packageSrc
  ```
</ParamField>

<ParamField path="packageDoc" type="TaskKey[File]">
  Produces a documentation artifact containing API documentation.
  
  ```bash
  sbt packageDoc
  ```
</ParamField>

### Documentation Tasks

<ParamField path="doc" type="TaskKey[File]">
  Generates API documentation using Scaladoc.
  
  ```bash
  sbt doc
  ```
</ParamField>

## Compiler Settings

<ParamField path="scalacOptions" type="TaskKey[Seq[String]]">
  Options for the Scala compiler.
  
  ```scala
  scalacOptions ++= Seq(
    "-deprecation",
    "-feature",
    "-unchecked"
  )
  ```
</ParamField>

<ParamField path="javacOptions" type="TaskKey[Seq[String]]">
  Options for the Java compiler.
  
  ```scala
  javacOptions ++= Seq("-source", "11", "-target", "11")
  ```
</ParamField>

<ParamField path="maxErrors" type="SettingKey[Int]" default="100">
  The maximum number of errors, such as compile errors, to list.
  
  ```scala
  maxErrors := 20
  ```
</ParamField>

## Dependency Management Settings

<ParamField path="libraryDependencies" type="SettingKey[Seq[ModuleID]]">
  Declares managed dependencies.
  
  ```scala
  libraryDependencies ++= Seq(
    "org.typelevel" %% "cats-core" % "2.10.0",
    "org.scalatest" %% "scalatest" % "3.2.17" % Test
  )
  ```
</ParamField>

<ParamField path="resolvers" type="SettingKey[Seq[Resolver]]">
  Defines additional resolvers for retrieving dependencies.
  
  ```scala
  resolvers += "Sonatype OSS Snapshots" at 
    "https://oss.sonatype.org/content/repositories/snapshots"
  ```
</ParamField>

## Runtime Settings

<ParamField path="fork" type="SettingKey[Boolean]" default="false">
  If true, forks a new JVM when running. If false, runs in the same JVM as the build.
  
  ```scala
  fork := true
  ```
</ParamField>

<ParamField path="javaOptions" type="TaskKey[Seq[String]]">
  Options passed to a new JVM when forking.
  
  ```scala
  javaOptions ++= Seq("-Xmx2G", "-Xms512M")
  ```
</ParamField>

<ParamField path="javaHome" type="SettingKey[Option[File]]">
  Selects the Java installation used for compiling and forking. If None, uses the Java installation running the build.
  
  ```scala
  javaHome := Some(file("/usr/lib/jvm/java-17"))
  ```
</ParamField>

## Common Patterns

### Scoping Keys

Keys can be scoped to different contexts:

```scala
// Scope to configuration
Test / sourceDirectory := baseDirectory.value / "test-sources"

// Scope to task
compile / scalacOptions += "-Ylog-classpath"

// Multiple scopes
Test / console / scalacOptions -= "-Ywarn-unused"
```

### Modifying Settings

```scala
// Set value
name := "my-project"

// Append to sequence
scalacOptions += "-deprecation"

// Append multiple items
scalacOptions ++= Seq("-feature", "-unchecked")

// Transform existing value
version := (version.value + "-SNAPSHOT")
```

<Note>
  Use `.value` to reference the value of another setting or task within a setting/task definition.
</Note>

### Task Dependencies

Tasks automatically track their dependencies:

```scala
customTask := {
  val classes = (Compile / compile).value  // Depends on compile
  val cp = (Compile / fullClasspath).value // Depends on classpath
  // Your task logic here
}
```

<Warning>
  Don't use `.value` inside conditionals or lambdas - it must be called at the top level of the task body.
</Warning>
